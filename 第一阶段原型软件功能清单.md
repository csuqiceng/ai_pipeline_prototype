# 第一阶段原型软件功能清单

## 1. 文档目的

本文档用于明确“二手机械手 AI 智能化升级改造项目”第一阶段原型软件的功能范围，帮助研发团队聚焦最小可行系统，避免前期功能过多、开发失控。

第一阶段目标不是做完整工业智能机器人平台，而是做出一套可运行、可演示、可联调、可试点的原型软件系统。

## 2. 第一阶段软件目标

第一阶段软件需要完成以下核心目标：

- 建立工控机上位机主控程序
- 接入 Motion SDK 并能够控制 ZMC406
- 提供基础 UI 界面
- 支持固定动作调用
- 支持基础语音命令触发
- 支持基础视觉识别结果接入
- 支持基础任务调度与状态监控
- 支持报警、急停和运行日志

## 3. 第一阶段功能边界

### 3.1 本阶段必须实现

- 手动控制
- 固定动作调用
- 控制器连接与状态显示
- 基础日志记录
- 基础报警处理
- 简单语音命令识别接入
- 简单视觉结果接入
- 一个完整演示任务闭环

### 3.2 本阶段暂不强求

- 复杂自由对话
- 多目标最优路径规划
- 高精度实时轨迹插补联动
- 连续视觉闭环修正
- 多机械手协同
- 云端大规模任务调度

## 4. 第一阶段原型软件模块清单

### 4.1 系统主控模块

功能说明：

- 启动软件主程序
- 初始化配置
- 初始化各子模块
- 管理程序运行状态

核心功能：

- 软件启动与退出
- 模块初始化
- 配置文件加载
- 系统状态管理

验收标准：

- 软件可正常启动
- 模块初始化状态可见
- 配置错误时可提示

### 4.2 控制器连接管理模块

功能说明：

- 负责上位机与 ZMC406 的连接、断开、重连和状态监控

核心功能：

- 连接控制器
- 断开控制器
- 自动/手动重连
- 读取连接状态
- 读取基础控制器信息

验收标准：

- 可稳定建立连接
- 掉线可提示
- 连接状态界面可见

### 4.3 Motion SDK 封装模块

功能说明：

- 对官方 SDK 做项目级二次封装，供上位机其他模块统一调用

核心功能：

- SDK 初始化
- 动作调用封装
- 状态读取封装
- 错误码处理
- 异常返回处理

建议接口：

- `connect_controller()`
- `disconnect_controller()`
- `home()`
- `stop()`
- `reset()`
- `run_action(action_name, params)`
- `get_status()`

验收标准：

- 上位机可通过统一接口调用控制功能
- 错误返回可记录

### 4.4 手动控制模块

功能说明：

- 用于调试和演示阶段的人工控制

核心功能：

- 回零
- 开始
- 停止
- 急停
- 抓取
- 放置
- 搬运
- 复位

验收标准：

- 页面按钮可直接触发机械手动作
- 动作执行结果可显示

### 4.5 自动任务执行模块

功能说明：

- 接收结构化任务并触发预设动作

核心功能：

- 接收任务 JSON
- 任务解析
- 动作映射
- 启动任务
- 任务完成状态回传

第一阶段特点：

- 只支持固定任务类型
- 优先使用固定动作库，不做复杂自由轨迹规划

验收标准：

- 可接收标准任务
- 能自动执行一个完整任务流程

### 4.6 状态监控模块

功能说明：

- 显示系统当前运行状态

核心功能：

- 控制器连接状态
- 机械手运行状态
- 当前任务状态
- 报警状态
- 急停状态

验收标准：

- 状态能实时更新
- 关键异常能被界面感知

### 4.7 日志模块

功能说明：

- 记录关键运行过程，便于联调、排障和后期追踪

核心功能：

- 启动日志
- 连接日志
- 指令发送日志
- 动作执行日志
- 报警日志
- 异常日志

验收标准：

- 关键事件可追踪
- 出错时有日志可查

### 4.8 报警与安全模块

功能说明：

- 保证第一阶段演示和调试安全

核心功能：

- 急停按钮联动
- 控制器异常报警提示
- 安全区异常禁止执行
- 非法状态禁止执行

验收标准：

- 急停可以立即打断任务
- 报警信息可显示

### 4.9 视觉接入模块

功能说明：

- 接收视觉模块输出的基础结果

第一阶段功能：

- 接收目标是否存在
- 接收目标基础位置
- 接收安全区状态
- 将结果显示在 UI

验收标准：

- 能接收视觉结果
- 能基于视觉结果触发动作

### 4.10 语音接入模块

功能说明：

- 接收语音识别模块输出的命令结果

第一阶段功能：

- 接收固定命令词
- 接收语义意图
- 转换为固定动作任务

建议支持命令：

- 开始作业
- 停止作业
- 回零
- 抓取目标
- 放置目标
- 复位

验收标准：

- 固定命令可驱动任务执行

### 4.11 视频显示模块

功能说明：

- 在 UI 中显示摄像头画面和基础识别结果

核心功能：

- 显示视频流
- 显示识别框或识别状态
- 显示目标存在与否

验收标准：

- 视频画面可稳定显示
- 基础识别信息可叠加显示

### 4.12 参数配置模块

功能说明：

- 管理系统基础参数

第一阶段建议参数：

- 控制器连接参数
- 动作速度参数
- 语音开关参数
- 摄像头参数
- 日志路径

验收标准：

- 参数可读取和保存
- 修改后可生效

## 5. 第一阶段建议 UI 页面

建议最少包含以下页面：

- 首页状态总览
- 手动控制页面
- 自动任务页面
- 视频监控页面
- 报警日志页面
- 参数配置页面

## 6. 第一阶段建议任务类型

建议只支持以下任务：

- `go_home`
- `pick`
- `place`
- `pick_and_place`
- `stop_task`
- `reset_robot`

## 7. 第一阶段建议数据流

```text
语音命令 / 视觉结果
    ->
结构化任务
    ->
上位机任务解析
    ->
Motion SDK 调用
    ->
ZMC406 执行动作
    ->
状态返回 UI
```

## 8. 第一阶段最低验收标准

- 软件可启动、可关闭
- 可连接 ZMC406
- 可手动回零、抓取、放置、停止
- 可显示控制器状态
- 可记录日志
- 可接收视觉结果
- 可接收语音命令
- 可完成一次完整抓取搬运演示
- 急停与异常提示可用

## 9. 开发建议

- 第一阶段优先做“稳定”和“可控”
- 不要在首版加入过多智能功能
- 所有 AI 输出必须转成结构化任务
- 所有危险动作必须经过安全检查
- 所有异常必须记录日志

## 10. 结论

第一阶段原型软件的核心任务是搭建一套能够稳定跑通“语音/视觉输入 -> 上位机调度 -> Motion SDK 通讯 -> ZMC406 执行 -> 状态反馈”的最小闭环系统。只要这个闭环打通，后续视觉精定位、轨迹规划和大模型增强都可以在此基础上持续迭代。
