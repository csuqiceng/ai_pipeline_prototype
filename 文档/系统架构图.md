## 2. 控制权归属

按“谁真正让机械手动起来”来分：

- `AI/上位机`：负责“想干什么”
- `Motion SDK + ZMC406`：负责“把命令送到控制器”
- `ZBasic + 控制器程序`：负责“怎么安全地去做”
- `机械手硬件`：负责“实际执行动作”

也就是说：

```text
AI / Python 上位机
    负责：任务理解、任务编排、界面、协议、联调入口

ZMC406 + ZBasic
    负责：运动控制、回零、暂停、继续、IO、报警、安全逻辑

机械手本体
    负责：真实执行
```

## 3. 你当前所在的位置

你现在的角色更接近：

- 上位机原型开发
- AI 结果接入与任务编排
- 控制器联调接口层

你现在不是：

- 机械臂底层控制器主程序作者
- 伺服调试工程师
- 电气安全回路负责人

一句话概括：

`你在做的是“让 AI / 上位机 能和控制器那套现成的机械手控制逻辑正确对接”。`

## 4. 当前仓库在系统中的位置

当前仓库主要覆盖这部分：

```text
语音/视觉输入
    ↓
任务理解与 JSON 生成
    ↓
任务调度器
    ↓
执行器接口
    ↓
控制器服务
    ↓
Motion SDK 适配
```

对应文件：

- `ai_pipeline_prototype/inputs.py`
- `ai_pipeline_prototype/planner.py`
- `ai_pipeline_prototype/dispatcher.py`
- `ai_pipeline_prototype/executor.py`
- `ai_pipeline_prototype/controller_service.py`
- `ai_pipeline_prototype/sdk_adapter.py`

这套代码当前已经能做：

- 模拟模式演示
- SDK 风格调用封装
- 控制器连接、状态刷新、回零、停止的联调入口

这套代码当前还没有完全覆盖：

- 真机稳定动作闭环
- 控制器内全部工艺流程
- 完整的现场安全联锁
- 完整的报警处理与自动化测试

```text

```

## 6. 你应该和谁联调

当前最关键的联调对象不是 AI 同学，而是控制器 / 自动化侧。

### 6.1 第一优先级：控制器 / 自动化工程师

需要和他确认：

- 当前现场到底是“控制器内置 ZBasic 主控”还是“PC 直接 SDK 点位控制”
- 轴号定义
- 回零模式
- 停止模式
- 夹具输出口
- 哪些寄存器 / IO / 报警码代表什么
- 上位机应该发“点位”还是发“动作触发”

### 6.2 第二优先级：现场设备负责人

需要和他确认：

- 机械手当前是否可安全联调
- 伺服是否上电
- 急停、限位、安全开关是否正常
- 当前现场是否允许回零和小范围动作测试

### 6.3 第三优先级：AI / 视觉 / 语音同学

需要和他们确认：

- 输入协议
- JSON 字段
- 置信度和安全区输出

但这不是当前最核心的真机联调对象。
